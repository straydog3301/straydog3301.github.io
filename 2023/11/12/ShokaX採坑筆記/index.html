第一次使用 Hexo 在建置時遇到了不少狀況 甚至踩了第一天環境弄好躍躍欲試的想開始測試但怎樣編譯網站都出不來，弄了好幾個小時無果結果隔天就好了，估計是 node.js 裝好要先重開機才能正常運作的坑

這邊就只針對使用 ShokaX 所遭遇的狀況做個紀錄

使用的主題 https://github.com/theme-shoka-x/hexo-theme-shokaX

# 前言

之所以使用這個主題是因為在 Hexo 官網亂翻看到覺得這個主題設計的很好看 再來就是有中文文件我才看得懂 用下來感覺還是很不錯的，很多功能都內置好了不用自行開發，不過個人建置下遇到了一些狀況所以記錄下來也算是給自己初次寫 blog 的練習

# 配置文件

最初的導覽列上有個友連的頁面連結但是我沒有朋友 照著 github 說明上設置_config.shokax.yml 卻怎樣都弄不掉，後來發現配置文件內容只能覆寫沒辦法直接刪除，因為我很懶所以土法煉鋼直接開 node_modules 資料夾找出 hexo-theme-shokaX 把作者的設置全註解掉複製到使用者的設置文件上再進行調整

# Icon 圖標

由於 ShokaX 是基於 shoka 的二次開發版，原 shoka 的開發者以多年沒有更新維護，原作者是使用阿里巴巴的 iconfont 來配置網站使用的圖標，一開始先是試著直接改控制 social 的代碼把 iconfont 改成用 img 顯示，不過效果不太好，但由於我也不太好意思聯繫原作者所以沒辦法基於原始的 icon 配置進行添加，只好自己在 iconfont 網站建立新專案設置一份所有 icon 都同名的 iconfont.styl 文件後在進行添加 (只要網站有使用到的 iconfont 幾乎都要寫進文件不然會直接報錯沒辦法編譯)，因為圖標太多我又太懶對照原作者用了哪些圖標，所以會有很多圖標不是原作者使用的狀況，不過總算是可以自由添加自己想用的 icon 啦 其實搞這一切只是為了弄一個噗浪圖示在頭像底下

# 留言區

大概是搞最久的地方了，一開始還想說乾脆把噗浪直接崁入進來，不過太人工了 其實外崁的噗浪會直接取消登入狀態所以也沒辦法正常顯示或留言 後來選擇了主題內建好的幾套留言系統中挑了 waline，因為 waline 有很棒的功能是可以自訂表符，但版權問題也不好直接搬噗浪的表符過來我想念狐狐，有空大概會在自己慢慢畫一些自爽用的小表符進來 (つд⊂)
不過還是有不少問題所以又改了幾段 code

# 1. 最新留言區超連結判定錯誤

這個問題不太確定是不是我配置動到了什麼東西，最新留言跑出的超連結網域永遠是 0.0.7.231 我還跑去查這到底是哪裡的 ip
pug 和 veu 我都不熟所以看了很久其實我根本不會寫程式

在來看 asset.js this.config.url 是我設置在_config 的 github.io 沒問題，但 replace 處理後 root 竟然存了個空字串我看不懂正規表達式所以就直接結果論

試著把 root 改成我的 github.io 後發現網域和子連結中間會有兩個斜槓

像這樣 https://straydog3301.github.io//2023/11/02/My-New-Post/#654ef6f135a11e14ba3984e2

這樣的連結還是沒辦法導向正確的留言位置，但如果是空字串的時候就不會帶雙斜槓，為了瞭解這個神秘的現象我又繼續往下看，看到了作者對留言超連結是這樣處理的

我不知道甚麼狀況下留言連結會 == ‘/’ 所以就預設 url 一定會加上斜槓好了，這不是我預期的效果所以我就直接把這句砍了
後來發現我這樣處理只對在首頁時有效，如果我在文章下點擊最新留言連結依舊會跑掉
目前先列入代辦清單

# 2. 最新留言的內容會顯示 html 標籤

因為我很愛用表符，所以自己留了幾個言後發現預覽內容如果都是 <img src=“…”> 根本看不出到底留了什麼
所以問了 GPT 大神要怎麼濾掉 html 標籤，感謝偉大的 ChatGPT 教我寫程式

# 3. 預設頭像很醜

雖然有找到替換預設頭像的方法，不過感覺大家頭像都一樣還是太單調了
目前先寫了個小工具隨機刷新頭像顯示 (從我做的小表符裡面挑) 不過寫法有點暴力之後在找時間優化吧

# Banner

文章內的 cover 沒辦法設置本地圖片，直接用開發者模式看發現圖片路徑都導到文章連結下，但如果針對每個文章都開資料夾放一張封面圖感覺超級麻煩所以決定要設置封面圖就通通外連好了。

# 畫廊

其實這也不算坑，Fancybox 已經有在配置文件中被標示不可用，但姑且測試了一下發現是縮圖跑不出來但幻燈片的功能是沒問題的，不過我有放圖庫的需求，這個有必要修看看或尋找替代方案 (´;ω;`)
